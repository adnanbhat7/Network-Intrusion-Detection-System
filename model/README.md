# model/

Place your trained model files here before running the website.

## Required files

| File | Description | Generated by |
|---|---|---|
| `cnn_model.keras` | Trained 1D-CNN | `cic_nids_cnn.ipynb` Cell 14 |
| `scaler.pkl` | Fitted RobustScaler | `cic_nids_cnn.ipynb` Cell 14 |
| `label_encoder.pkl` | Fitted LabelEncoder | `cic_nids_cnn.ipynb` Cell 14 |
| `features.pkl` | Ordered feature name list | `cic_nids_cnn.ipynb` Cell 14 |

## How to generate

1. Run `cic_nids_cnn.ipynb` in Colab from top to bottom.
2. Cell 14 saves four files automatically with these names:
   - `cic_ids_cnn_final.keras`
   - `cic_ids_cnn_scaler.pkl`
   - `cic_ids_cnn_labelencoder.pkl`
   - `cic_ids_cnn_features.pkl`

3. Copy and rename them into this folder:

```bash
cp cic_ids_cnn_final.keras      shieldnet/model/cnn_model.keras
cp cic_ids_cnn_scaler.pkl       shieldnet/model/scaler.pkl
cp cic_ids_cnn_labelencoder.pkl shieldnet/model/label_encoder.pkl
cp cic_ids_cnn_features.pkl     shieldnet/model/features.pkl
```

## What happens without the files

The website still runs. The simulator uses profile-based simulation
and the AI analysis text is still generated by Claude.
The `/api/predict` endpoint returns HTTP 503 and the frontend falls back
gracefully — no crash, no blank screen.

## Inference flow

```
POST /api/predict  { features: { "FLOW DURATION": ..., "FWD PKT/S": ..., ... } }

  → loader.py builds feature vector in correct column order
  → RobustScaler normalises
  → reshape to (1, n_features, 1) for Conv1D
  → model.predict() → softmax probabilities
  → return { label, confidence, probabilities }
```
